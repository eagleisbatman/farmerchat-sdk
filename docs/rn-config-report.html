#1
Module not found — SDK source unreachable without build step
packages/react-native/package.json · Metro bundler
Error
Unable to resolve module @digitalgreenorg/farmerchat-react-native
None of these files exist:
  * node_modules/@digitalgreenorg/farmerchat-react-native/dist/index.js
  * node_modules/@digitalgreenorg/farmerchat-react-native/dist/index.js.js
Root Cause
The package's main and exports fields both point to ./dist/, which is the compiled output directory. During development in a monorepo, dist/ does not exist until an explicit build step (pnpm turbo build) is run. Metro — Expo's bundler — reads the "source" field (a Metro convention) to resolve the raw TypeScript source instead, bypassing the need for a prior build.
Fix — Added "source" field to package.json
// packages/react-native/package.json — before
{
  "main":  "./dist/index.js",
  "types": "./dist/index.d.ts",
  ...
}

// after
{
  "main":   "./dist/index.js",
  "source": "./src/index.ts",  ← Metro reads this in dev; dist/ used in production
  "types":  "./dist/index.d.ts",
  ...
}
How it works
Metro's resolver checks for a "source" field before falling back to "main". In development, Metro compiles src/index.ts on-the-fly via Babel. In a production build (expo export), the resolved "main" (dist/) is used instead, pointing to the pre-built optimised output.
#2
pnpm symlinks not resolved — Metro cannot find workspace packages
apps/demo-rn · metro.config.js (missing)
Error
error: Error: ENOENT: no such file or directory, open
  '.../demo-rn/node_modules/@digitalgreenorg/farmerchat-react-native/src/index.ts'

or

error: Unable to resolve module @digitalgreenorg/farmerchat-core from
  '.../packages/react-native/src/FarmerChat.tsx'
Root Cause
pnpm uses a virtual store and symlinks to link workspace packages. Metro's default configuration does not:
  • Watch directories outside the project root (apps/demo-rn/)
  • Follow symlinks when resolving modules
  • Look for node_modules in the monorepo root
Without explicit configuration, Metro cannot find packages/react-native/ or its own dependency packages/core/, even though pnpm has correctly installed them.
Fix — Created metro.config.js
// apps/demo-rn/metro.config.js (new file)
const { getDefaultConfig } = require('expo/metro-config');
const path = require('path');

const projectRoot  = __dirname;
const workspaceRoot = path.resolve(projectRoot, '../..');

const config = getDefaultConfig(projectRoot);

// 1. Watch entire monorepo so Metro sees changes in workspace packages.
config.watchFolders = [workspaceRoot];

// 2. Resolve from both project and monorepo root node_modules.
config.resolver.nodeModulesPaths = [
  path.resolve(projectRoot,  'node_modules'),
  path.resolve(workspaceRoot, 'node_modules'),
];

// 3. Follow pnpm symlinks correctly.
config.resolver.unstable_enableSymlinks = true;

module.exports = config;
What each setting does
  • watchFolders — tells Metro to watch the entire monorepo root, so file changes in packages/react-native/src/ trigger hot reload in the demo app.
  • nodeModulesPaths — Metro searches both locations for node_modules, finding root-level dependencies (hoisted by pnpm) and demo-app-local ones.
  • unstable_enableSymlinks — enables symlink traversal so pnpm's virtual store links are followed when resolving modules.
#3
Missing New Architecture flag, permissions & expo-location plugin
apps/demo-rn/app.json · apps/demo-rn/package.json
Problems
  • No newArchEnabled — React Native 0.83 defaults to the New Architecture (Fabric + JSI). Without the explicit flag, Expo shows a deprecation warning and the build may fall back to the legacy bridge inconsistently.
  • No location permissions — the Onboarding screen requests ACCESS_FINE_LOCATION. Without the permission declaration the OS denies the request silently on Android and crashes with a missing usage description on iOS.
  • No microphone permissions — the voice input feature requires NSMicrophoneUsageDescription (iOS) and RECORD_AUDIO (Android) or the app is rejected from stores.
  • Missing expo-location — referenced in the plugins array but not declared as a dependency, causing expo prebuild to fail.
Fix — Updated app.json
{
  "expo": {
    "name": "FarmerChat Demo",
    "newArchEnabled": true,          ← New Architecture (RN 0.83)
    "ios": {
      "bundleIdentifier": "org.digitalgreen.farmerchat.demo",
      "infoPlist": {
        "NSLocationWhenInUseUsageDescription": "FarmerChat uses your location...",
        "NSMicrophoneUsageDescription": "FarmerChat uses the microphone..."
      }
    },
    "android": {
      "package": "org.digitalgreen.farmerchat.demo.rn",
      "permissions": [
        "ACCESS_FINE_LOCATION", "ACCESS_COARSE_LOCATION", "RECORD_AUDIO"
      ]
    },
    "plugins": [
      "expo-router",
      ["expo-location", { "locationWhenInUsePermission": "..." }]  ← added
    ]
  }
}
Fix — Added expo-location to package.json
// apps/demo-rn/package.json — dependencies
"expo-location": "~18.1.0",   ← added (compatible with Expo 55)
#4
SplashModule crash — expo-splash-screen, expo-constants & expo-linking missing from dependencies
apps/demo-rn/package.json · Runtime (iOS Simulator)
Errors
Error [ERR_STREAM_UNABLE_TO_PIPE]: Cannot pipe to a closed or destroyed stream

 ERROR  [Error: Uncaught (in promise, id: 0):
   "TypeError: SplashModule.internalPreventAutoHideAsync is not a function (it is undefined)"]

 ERROR  [Error: Uncaught (in promise, id: 1):
   "TypeError: SplashModule.internalMaybeHideAsync is not a function (it is undefined)"]
Root Cause
Expo SDK 55 changed its package versioning scheme — all first-party Expo packages now use the SDK major version number (~55.x.x instead of the old independent version numbers like ~0.29.x). Three packages required by expo-router were not declared in package.json:
  • expo-splash-screen — manages the native splash screen that hides after the app JS finishes loading. When absent, pnpm resolves an incompatible transitive version whose native module (SplashModule) exports different function names than the JS wrapper expects, causing both errors above.
  • expo-constants — provides app.json values at runtime. Required by expo-router for app config access.
  • expo-linking — deep linking API. Required by expo-router for URL-based navigation.
The ERR_STREAM_UNABLE_TO_PIPE error is a cascade — Metro's dev server pipe breaks when the native runtime crashes on startup.
Fix — Added all missing Expo 55 packages to package.json
// apps/demo-rn/package.json — dependencies (before → after)

Before:
"expo": "~55.0.0",
"expo-location": "~18.1.0",      ← wrong version format for SDK 55
"expo-router": "~5.0.0",
"expo-status-bar": "~3.0.0",   ← wrong version format for SDK 55
// expo-splash-screen missing ← CRASH
// expo-constants missing
// expo-linking missing

After:
"expo": "~55.0.0",
"expo-constants":    "~55.0.0",  ← added (SDK 55 versioning)
"expo-linking":      "~55.0.0",  ← added (SDK 55 versioning)
"expo-location":     "~55.0.0",  ← corrected to SDK 55 versioning
"expo-router": "~5.0.0",
"expo-splash-screen": "~55.0.0", ← added — fixes the crash ✓
"expo-status-bar":   "~55.0.0",  ← corrected to SDK 55 versioning
After applying the fix
Run these commands to reinstall and restart with a clean cache: 
# From the monorepo root
pnpm install

# Then start with Metro cache cleared
cd apps/demo-rn
pnpm start -- --clear

How to Run the React Native Demo
⚠️
Prerequisites: Node.js 22+, pnpm 10+, Xcode (for iOS Simulator), Android Studio with an AVD (for Android Emulator).
  • 1
    Install pnpm (if not installed): 
    npm install -g pnpm@10
  • 2
    Pull latest code & install all workspace dependencies from the monorepo root: 
    cd /Users/Aiyappa/AndroidStudioProjects/farmerchat-sdk
git pull origin feature/android-studio-config
pnpm install
  • 3
    Set your API key (optional — falls back to "demo-key"): 
    echo 'EXPO_PUBLIC_FC_API_KEY=your-real-key' > apps/demo-rn/.env.local
  • 4
    Start the Expo dev server: 
    cd apps/demo-rn
pnpm start
  • 5
    In the terminal, press i for iOS Simulator, a for Android Emulator, or scan the QR code with Expo Go on a real device.
  • 6
    (Optional — production build only) Build the SDK package first so dist/ is available: 
    pnpm turbo build --filter=@digitalgreenorg/farmerchat-react-native...
✅
For development, no build step is needed. Metro resolves packages/react-native/src/index.ts directly via the "source" field. Hot reload works across the entire monorepo.
